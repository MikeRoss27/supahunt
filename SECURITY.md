# Security Policy

## Responsible Use

SupaHunt is designed for **authorized security testing only**. This includes:

- Bug bounty programs where Supabase applications are in scope
- Penetration testing engagements with written authorization
- Security auditing of your own applications
- CTF challenges and security research

## What This Tool Is NOT For

- Unauthorized access to systems you do not own or have permission to test
- Attacking production systems without explicit written consent
- Data theft, service disruption, or any malicious activity

## Reporting Vulnerabilities in SupaHunt

If you find a security issue in SupaHunt itself (e.g., a dependency vulnerability, code injection in report generation, credential leak in output files), please:

1. **Do NOT open a public issue**
2. Email: Open a private security advisory via GitHub's "Report a vulnerability" feature on this repo
3. Include: description, reproduction steps, and impact assessment

We will respond within 48 hours.

## Reporting Supabase Vulnerabilities

If you use SupaHunt and discover vulnerabilities in the Supabase platform itself (not a specific application), report them to:

- Supabase: https://supabase.com/.well-known/security.txt
- HackerOne: https://hackerone.com/supabase (if applicable)

## Legal

Users are solely responsible for ensuring they have proper authorization before using this tool. The authors assume no liability for misuse.
